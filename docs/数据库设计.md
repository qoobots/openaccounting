# 企业会计系统 - 数据库设计文档

## 1. 数据库概述

**数据库类型**: MySQL 8.4  
**字符集**: utf8mb4  
**排序规则**: utf8mb4_general_ci  
**命名规范**:
- 表名: 小写字母+下划线，如 `sys_user`
- 字段名: 小写字母+下划线，如 `user_name`
- 索引名: `idx_表名_字段名`

## 2. 核心实体关系图

```
公司(sys_company)
    │
    ├─── 部门(sys_department) ──── 员工(sys_user)
    │                            │
    │                            └─── 用户角色关联(sys_user_role)
    │                                 │
    │                                 └─── 角色(sys_role) ──── 角色权限关联(sys_role_permission)
    │                                                            │
    │                                                            └─── 权限(sys_permission)
    │
    ├─── 科目(gl_account)
    │         │
    │         ├─── 凭证分录(gl_voucher_entry) ──── 凭证(gl_voucher)
    │         │                                    │
    │         └─── 总账余额(gl_balance)
    │
    ├─── 客户(ba_customer) ──── 往来明细
    ├─── 供应商(ba_supplier) ──── 往来明细
    ├─── 项目(ba_project) ──────── 辅助核算明细
    │
    └─── 资产(fa_asset) ────────── 资产折旧(fa_depreciation)
```

## 4. 数据库设计原则

### 4.1 命名规范
- 表名使用小写字母和下划线分隔
- 字段名使用小写字母和下划线分隔
- 主键统一使用雪花ID（BIGINT）
- 所有表包含公共字段：create_time, update_time, is_deleted

### 4.2 字段类型选择
- 金额字段使用 DECIMAL(18,2)
- 日期时间使用 DATETIME
- 日期使用 DATE
- 状态字段使用 TINYINT 或 VARCHAR
- 文本字段根据长度选择 VARCHAR 或 TEXT

### 4.3 索引设计原则
- 主键索引：所有表都有主键
- 唯一索引：业务唯一字段
- 普通索引：高频查询字段
- 复合索引：多字段联合查询
- 索引名称格式：idx_表名_字段名

### 4.4 约束设计
- 外键约束：不使用物理外键，通过应用层维护
- 非空约束：关键字段设置 NOT NULL
- 默认值：为常用字段设置默认值

---
