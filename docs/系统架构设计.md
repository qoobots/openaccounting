# 企业会计系统 - 架构设计文档

## 1. 业务架构设计

### 1.1 业务领域划分

#### 核心业务域
| 业务域 | 描述 | 核心能力 |
|--------|------|----------|
| 基础数据 | 组织、人员、往来单位等基础档案 | 档案管理、主数据维护 |
| 总账核算 | 科目、凭证、账簿 | 凭证处理、账簿查询、期末处理 |
| 辅助核算 | 部门、项目、往来等辅助核算 | 多维度核算、辅助账查询 |
| 财务报表 | 各类财务报表 | 报表生成、报表导出 |
| 预算管理 | 预算编制、执行、控制 | 预算编制、预算控制、预算分析 |
| 固定资产 | 资产卡片、折旧 | 资产管理、折旧计算 |
| 审批流程 | 各类业务审批 | 工作流引擎、审批任务 |

### 1.2 业务流程架构

#### 核心业务流程

```
会计核算主流程:
原始单据 → 凭证编制 → 凭证审核 → 凭证过账 → 期末处理 → 财务报表
    ↓         ↓         ↓         ↓          ↓           ↓
  发票      录入分录   审批流程   生成分录   月结/年结   报表生成
```

#### 详细业务流程

**1. 凭证处理流程**
```
业务发生 → 选择凭证模板 → 录入凭证分录 → 借贷平衡检查 → 
暂存凭证 → 提交审核 → 审核通过 → 凭证过账 → 生成账簿数据
                                    ↓
                              审核不通过 → 退回修改 → 重新提交
```

**2. 期末结账流程**
```
结账前检查(凭证全部审核、银行对账完成) → 
自动转账(损益结转、成本结转) → 
试算平衡 → 
期间结账(不可再录入凭证) → 
生成报表 → 
结账完成
```

**3. 预算管理流程**
```
预算年度开启 → 编制预算 → 预算审批 → 预算执行 → 
执行监控 → 差异分析 → 预算调整 → 年度预算关闭
```

### 1.4 业务规则

#### 科目规则
- 科目编码采用4-2-2-2结构，支持4级
- 科目余额方向：借方/贷方
- 科目属性：资产类、负债类、所有者权益类、成本类、损益类
- 科目辅助核算：部门、项目、往来单位、员工、自定义项

#### 凭证规则
- 凭证号规则：年度+期间+流水号 (如：2026010001)
- 每张凭证至少2条分录
- 借贷必须平衡
- 已审核凭证不可修改
- 已过账凭证不可删除

#### 结账规则
- 当月凭证必须全部审核
- 银行对账必须完成
- 损益类科目必须结清
- 结账后不能再录入当月凭证

### 2.4 应用服务接口设计

#### RESTful API 规范

**命名规范**
- 资源使用名词复数：`/accounts`, `/vouchers`
- 使用HTTP动词表示操作：GET(查询)、POST(新增)、PUT(修改)、DELETE(删除)

**统一响应结构**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1706112000000,
  "traceId": "uuid"
}
```

### 4.3 核心技术设计

#### 4.3.1 认证授权设计

**认证方式**
- 用户名密码登录
- JWT Token认证
- Token刷新机制
- 单点登录支持

**授权模型**
```
用户(User) ←→ 角色(Role) ←→ 权限(Permission)
     ↑                            ↓
     └───────── 组织机构(Organization)
```

**权限控制**
- 基于角色的访问控制 (RBAC)
- 数据权限：公司级、部门级、个人级
- 接口权限：@PreAuthorize注解

#### 4.3.2 多租户设计

**租户隔离策略**
- 共享数据库，独立Schema
- 所有表增加tenant_id字段
- MyBatis Plus拦截器自动注入租户ID

**租户上下文**
```java
ThreadLocal<Long> tenantContext = new ThreadLocal<>();
```

#### 4.3.3 分布式事务

**事务类型**
- 本地事务：单服务内事务
- 分布式事务：跨服务事务

**解决方案**
- Seata AT模式
- 最终一致性：消息队列
- 补偿机制：定时任务检查

#### 4.3.4 缓存设计

**缓存策略**
- 本地缓存：Caffeine
- 分布式缓存：Redis
- 多级缓存：L1(Caffeine) + L2(Redis)

**缓存分类**
```
系统参数缓存 → 永久缓存
科目数据缓存 → 定时刷新
权限数据缓存 → 过期刷新
报表数据缓存 → 计算后缓存
```

#### 4.3.5 接口幂等性

**幂等性方案**
- 唯一ID校验：请求ID+业务标识
- Token机制：请求前获取Token
- 数据库唯一索引

**实现方式**
```
请求 → 获取幂等Token → 提交业务 → 验证Token → 执行业务
                                    ↓
                              Token已使用 → 返回之前结果
```
